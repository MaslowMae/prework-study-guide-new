HTML

  • We use HTML to build a sturdy frame, using HTML elements as our frames building blocks
  • The DOCTYPE html is a declaration that tells the browser what document type it should expect when it is opened
  • The opening tag is HTML which represents the root or start of the document
  • The lang attribute is found inside the opening tag which tells the browser the language for the page.
  • The head element contains information about the webpage such as metadata to render the page correcly and the title of the page for SEO
  • The body element represents the visible content shown to the user for them to also interact with
  • The section element is inside the body to provide organized thematic content
  • The header element is at the top of the body and can contain navigation links or menus
  • The footer element is at the bottom of the page and usually contains author, copyright, contacts, or sitemap

CSS

  • We use CSS to style of presentation for users, such as margins, colors, visuals, layout, and organization
  • The most common way to style a webpage is by using an external CSS style sheet. This is preferred for separation of concerns
  • Declarations contain two important components: the CSS property we want to apply and the value of the property. For example, the declaration color: blue
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The padding property is often confused with margin, but it is not the same thing. The padding property adds space around the content inside an element
  • The CSS box model: every HTML element is essentially a box and consists of margins outside the element, and the element's borders, padding, and content
  • (DRY) Do Not Repeat Yourself : Coding principle to reuse or share pieces of code to reduce the number of lines and overall size
  • To link our css file that is in the same location as a seperate file we need to use the HTML link element

Git

  • Git allows us to work locally and change our work via branching in a way that doesn't conflict with collaborators
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch
  • stage changes when done with git add -A
  • commit the changes with git commit -m "added blank"
  • git init: initiallizes an existing directory as a git repository
  • when in the correct directory in the terminal, type code . to open visual studio code workspace
  • an SSH key is an access credential for the secure shell network protocol and allow for access without repeated password input
  • it is important to properly create and share your ssh keys to allow developers to interface with remote services like git
    • the ssh keys are generated and then added to the ssh agent which keeps track of keys and passphrases

JavaScript

  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites
  • Using JavaScript, you can power your websites with effects, interactivity, data communication, and much more
  • JavaScript is the tool that takes a static HTML/CSS webpage and makes it interactive
  • JavaScript is case-sensitive and uses the Unicode character set
  • Variables are containers that store values. You start by declaring a variable with the let keyword, followed by the name you give to the variable:A variable is a named reference to a value
  • The console.log method was created to make it easier for developers to test their code. Developers use it daily to debug and test the code that they write
  • Control flow is the order in which a computer executes code in a script. In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page
  • Control flow can be interrupted, however. We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all
